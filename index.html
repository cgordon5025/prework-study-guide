HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • h1 is the largest heading size, heading size will decrease with ascending header numbers
  • Place the link for the style sheet within the head element
  • Place the link for script in the body section, after the footer

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element
  • You can state generally what color you want the font/background to be in (i.e. gray, white black etc.) or you can provide the 6 digiti hexadecimal code
  • Here you can establish a class and what attributes are given to everything that receive said class, then in the html you will assign class to each element
  • Remember to group things if you want every header type to be the same, or every image to have the same properties, this will save time in the long run

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and swithces to it
  • git push origin: pushes the changes of the branch to github to be later asked for pull requests
  • After push command go into Github to review the pull request
